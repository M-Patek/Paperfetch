# Security Policy

## Supported Versions

The repository is in `0.x` development. Security and stability fixes will be applied to the latest development line.

## Reporting A Vulnerability

- do not open public issues for suspected vulnerabilities
- report the issue privately to the maintainers once a repository security contact is configured
- include reproduction steps, impact, and affected versions when possible

Until dedicated security contacts are configured, use the most private maintainer contact path available on the repository host.
